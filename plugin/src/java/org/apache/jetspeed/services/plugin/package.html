Thrust

Jetspeed's Plugin Architecture

1. What is Thrust?

Thrust is a simple architecture that allows developers to write pluggable/configurable application modules based on standardized interface definitions that introduce a low level of coupling and a high level of re-useability.

2. What makes Thrust different than a service architecutre like Fulcrum?

Service frameworks, like fulcrum, allow for only single implementation of a specific interface.  Thrust can have any number of named plugin's that all implement the same interface.  Services generally provide a high-level business functionallity. Thrust modules should be small, and portable. Thrust lends itself well to things like abstracting security implementations and persistence mechanisms.  Thrust plugins would be used to support high-level functionallity provided services.
